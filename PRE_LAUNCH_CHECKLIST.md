# צ'ק ליסט בדיקות לפני העלאה לאוויר (Pre-Launch Testing Checklist)

רשימה מפורטת לבדיקת המערכת (Client + Server) לפני עלייה לפרודקשן.

## 🎨 UI / UX
- [ ] בדיקה ויזואלית לכל המסכים והרכיבים.
- [ ] יישור נכון של אלמנטים.
- [ ] אחידות צבעים, פונטים ומרווחים.
- [ ] אנימציות ומעברים פועלים בצורה חלקה ולא מעכבים את המשתמש.
- [ ] Feedback למשתמש בכל פעולה: דיאלוגים (Radix UI), spinners במצב טעינה.

## ⚙️ בדיקות פונקציונליות
- [ ] וידוא שכל הפעולות המרכזיות (CRUD: יצירה, קריאה, עדכון, מחיקה) עובדות ללא שגיאות ב-Frontend וב-Backend.
- [ ] התמודדות עם State גלובלי (בזמן איבוד רשת, טעינת דף מחדש).
- [ ] בדיקת API בשרת (Express) מול הלקוח.

## 🌐 תאימות דפדפנים
- [ ] Chrome (Windows/Mac/Android)
- [ ] Safari (Mac/iOS)
- [ ] Firefox
- [ ] Edge

## 📱 תאימות מובייל וטאבלט ורספונסיביות
- [ ] פריסת עמודים במצב מובייל (מסכים קטנים - 320px עד 480px).
- [ ] פריסת עמודים במצב טאבלט (מסכים בינוניים - 768px עד 1024px).
- [ ] אלמנטים ניתנים ללחיצה (Touch targets) גדולים מספיק.
- [ ] הסתרת אלמנטים כבדים במובייל במידת הצורך.

## 🚀 ביצועים (מהירות טעינה)
- [x] הרצת `npm run build` בVite ווידוא שה-Rollup מייצר Chunk-ים אופטימליים.
- [ ] Lazy loading לתמונות ולרכיבי React כבדים.
- [ ] שרשורים למסדי נתונים בשרת – וידוא אינדקסים (Indexes) ב-MongoDB להאצת שאילתות ב-Mongoose.
- [ ] כלי בדיקה: הרצת Lighthouse לבדיקת מהירות ויצירת דו"ח.

## 🔒 אבטחה
- [x] בדיקת CORS - וידוא שיש הרשאות קפדניות `cors({ origin: 'your-domain' })`.
- [x] פגיעות XSS ו-Injection: וידוא השימוש ב `express-mongo-sanitize` ושאין זריקת נתונים מסוכנים לקליינט.
- [x] ניהול סשנים/טוקנים מוגן והסתרת מידע רגיש (`dotenv` שמכיל Keys אמיתיים רק בשרת הפרודקשן).
- [x] Rate Limiting (למניעת התקפות Brute Force ו-DDOS).
- [x] הקפדה על HTTPS.

## ♿ נגישות (Accessibility)
- [ ] שימוש בתוויות הולמות (ARAI labels).
- [ ] ניווט במקלדת אפשרי לכל ה-Forms והכפתורים.
- [ ] קונטרסט צבעים תקין לקריאה.
- [ ] כל התמונות מכילות ערך `alt`.

## 🔍 SEO בסיסי
- [x] תגיות `title` ו-`meta description` מותאמות (הוספת React Helmet או הגדרות ישר ב-Vite).
- [ ] Open Graph tags לשיתוף נכון ברשתות חברתיות.
- [x] וידוא שקובץ `robots.txt` ומפת אתר (`sitemap.xml`) קיימים, במידה וזה פרויקט שנסרק על ידי מנועי חיפוש (נוסף `robots.txt` שחוסם סריקה במידה וזו אפליקציה פרטית).

## 🏋️ בדיקות עומס
- [ ] שימוש בכלים כמו Apache JMeter, k6 או Artillery לדמות עומס של מאות/אלפי משתמשים.
- [ ] ניטור צריכת זיכרון בשרת של אפליקצית ה-Node.js שלכם.

## 📋 בדיקת טפסים והתראות
- [ ] ולידציות בקליינט (למשל מילוי שדות חובה, פורמט אימייל שגוי).
- [ ] ולידציות משלימות ב-Backend למנוע שיבוש נתונים.
- [ ] הצגת התראות למשתמש כאשר פעולה מצליחה (Success toast) או נכשלת (Error toast).

## 🛒 בדיקת תהליכי הרשמה/רכישה
- [ ] ביצוע רישום מלא, התחברות (Login), התנתקות מוקדמת ושינוי סיסמה.
- [ ] אימות מול מייל או סמס (אם קיים התהליך במערכת).
- [ ] סליקה ושילוב מול צד שלישי: בדיקה ב-Sandbox והדמיית כרטיס נדחה או כרטיס שעבר בהצלחה.

## 🔗 בדיקת קישורים שבורים
- [ ] מעבר על כל דפי הפרויקט וחיפוש שגיאות 404 לקבצים (תמונות חסרות) או נתיבים פנימיים וחיצוניים (Internal/External Routing).

## 🖊️ בדיקת תוכן (שגיאות כתיב, ניסוח)
- [ ] קריאה חוזרת של הקופירייטינג בעמודים הראשיים (Landing pages, הודעות ב-UI).
- [ ] אחידות השפה (פנייה בלשון אחידה - זכר/נקבה/רבים מול המשתמש).

## 📊 בדיקת אנליטיקה ופיקסלים
- [ ] Google Analytics, Facebook Pixel, Mixpanel - וידוא שהטריגרים נורים רק באיוונטים הנכונים.
- [ ] וידוא שהם אינם חוסמים/מאטים את טעינת הדף יתר על המידה (רצוי לטעון אסינכרונית - `async/defer`).

- [ ] גיבוי מסד הנתונים מוגדר כל X שעות/ימים ואומת בניסוי שחזור הלכה למעשה (Restore test).

## 🚨 בדיקת הודעות שגיאה
